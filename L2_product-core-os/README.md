# L2: Product Core OS

> 想定読者: コア規約・強制手段を設計／改定するコントリビュータ。

**L2 は L1 バンドルが継続運用される（メンテナンス性が保たれる）ための強制基盤である。** ここに固定されたルールだけが、バンドルへ取り込まれて他プロジェクトでも再現可能になる。

## 役割

- 常に参照される一次ルールを固定し、揺らぎを防ぐ
- L1 バンドルが「立ち上げ後も自己改善し続けられる」根拠を提供する
- 強制手段（Hook など）で規約の遵守を機械的に担保する

## 含めるもの / 含めないもの

- 含める: 組織共通規約、チーム運用規約、強制手段（Hook など）
- 含めない: 学習結果ログ（L3）、再利用バンドル成果物（L1）

## 原則

**L2 で固まらないルールはバンドル（L1）に混入させない。** 試験中・暫定の運用は L3 に記録し、安定化してから L2 に昇格させる。これにより、L1 を受け取った他プロジェクトでルール崩壊が起きないことを保証する。

## 現状

既存の `org/` `teams/*/rules.md` `.githooks/` は段階的にこの領域へ整理する。
