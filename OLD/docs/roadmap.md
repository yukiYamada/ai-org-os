# Roadmap

> 想定読者: 導入検討者、コントリビュータ、意思決定者。「いつ何が来るか」を把握し、採用判断や貢献計画に活かしたい人。

## このロードマップの位置づけ

本ドキュメントは ai-org-os の**方向性と達成判定基準**を共有するためのものです。リリース日や機能の厳密な約束ではありません。市場・技術・運用上の発見によって順序・範囲は調整されます。確定済みの方針変更は `teams/*/memory/decisions.md` に記録されます。

判断の起点は [Product Vision](./product_vision.md) の3軸（Build / Maintain / Share）です。各フェーズの取り組みは、3軸のいずれかへの寄与を必ず持ちます。

## 直近フェーズ（〜2026 Q3）

L1 バンドルの実運用品質を引き上げ、運営原則を週次レビューで安定運用する段階です。

- **L1 v0.3**: `web-service-team-bundle-v0` の install.md / customize.md を、外部適用試行のフィードバックを反映して改訂する
- **Metrics 週次計測の安定運用**: 採用指標（Build / Maintain / Share）を週次でレビューし、ダッシュボードを定常化する
- **各種運用1週レビュー実施**: Vision-First / Reader-Aware 等の原則について「実際に守れたか」を1週単位で評価し、原則そのものを更新可能にする

**達成判定基準**: L1 v0.3 の install.md が実適用1件のフィードバックを反映して改訂済み。週次 Metrics が4週連続で欠測なく記録されている。

## 次フェーズ（〜2026 Q4）

L1 を「外部導入実績つきの v1.0」として固定し、第二弾バンドルの選定に着手する段階です。

- **L1 v1.0 リリース**: 外部導入実績を最低1件取得し、後方互換ベースラインを固定する
- **第二弾バンドル候補の選定**: web-service 以外のドメイン候補を3案程度に絞り込み、ICE スコア等で1案を選定する
- **テンプレ v2**: 各 L1 テンプレートを「コピペ可能」から「具体的な記入済み事例つき」へ昇格させる

**達成判定基準**: L1 v1.0 の外部適用が1件 PR またはレポートとして公開済み。第二弾バンドルの選定理由が `decisions.md` に記録されている。

## 中期（2027 H1）

L1 を単一バンドルから「複数ドメインのバンドル群」へ拡張する段階です。

- **異ドメイン向け L1 バンドル展開**: モバイル / データ基盤 / 社内 SaaS などから1〜2ドメインで第二弾バンドルを公開
- **メタバンドル**: 複数バンドルの組合せ（例: web + data 基盤）を1つの導入パスとして提供する設計を試作

**達成判定基準**: 第二弾バンドルが install.md とカスタマイズガイドつきで公開済み。メタバンドルの設計案が L2 規約として明文化されている。

## 長期（2027 H2 以降）

運営構造の外側、すなわち自動化とコミュニティ運営に踏み込む段階です。

- **自動化レイヤー検討**: CI 統合 / 軽量オーケストレーションを「現プロダクトの非ゴール解除」として検討開始する（着手判断は Vision 更新を伴う）
- **コミュニティ運営とコントリビュータ受入**: GitHub Issues 中心の現状から、Discussion / RFC プロセスへ段階的に拡張する

**達成判定基準**: 自動化レイヤーの採否が `decisions.md` に記録されている。外部コントリビュータからのマージ済み PR が累計3件以上。

## 現状やらないこと（明示）

以下は本ロードマップの対象外です。必要が生じた段階で Vision を更新し、改めて議論します。

- 実行ランタイム（オーケストレーター / 課金 / UI）の本体実装
- 特定 LLM ベンダー専用機能の組込み
- マーケットプレイス機能
