# Team Local Rules

<!-- CUSTOMIZE: pr-granularity | ルール1の "1 変更 1 目的" を自社の妥当な PR 粒度へ。例: 変更ファイル原則3以下、差分±200行など -->
<!-- CUSTOMIZE: review-scope | ルール2の "仕様更新なしで挙動変更しない" をプロダクトのリスク階層に応じて緩和/強化 -->
1. 1 変更 1 目的を原則とする。  
2. 仕様更新なしで挙動変更しない。  
3. PR には「何を / なぜ / どう検証したか」を必ず記載。  
4. 迷った場合は実装より先に `specs/` を更新。  
5. レトロ結果から毎回 1 つ以上の改善アクションを次サイクルへ持ち越す。
6. 各タスク完了時に、ボトムアップ改善提案を最低 1 件提出する。
7. 提案は「症状 / 原因仮説 / 最小変更案 / 影響範囲」で記述する。
8. 提案の採否（採択/保留/却下）は `teams/*/memory/decisions.md` に記録する。
