# L1: Product OS Bundles

> 想定読者: バンドルを受け取って自社へ適用する側、およびバンドルを設計・出荷する側のコントリビュータ。

**L1 はこのツールの直接的な商品（プロダクトの出力物）である。** 他者が自社用 AI 開発チームとして立ち上げ・運用・共有できる「移植可能な学習済みバンドル」を配置する。

## 各バンドルが含むべき構成

- `README.md`: バンドルの目的・適用先・前提条件
- `mission.md`: そのチームのミッションと判断軸
- `rules/`: 組織共通規約とチーム運用規約のスナップショット
- `roles/`: 役割定義（責務・権限・引き継ぎ条件）
- `workflow/`: 標準ワークフローと運用フロー
- `templates/`: 起票・レビュー・議事用テンプレ
- `INSTALL.md`: 移植手順（適用順序・差分マージ指針）

## 含めない / 含めるの境界

- 含めない: コア規約の本文（L2 にのみ存在し、バンドルはそのスナップショットを取り込む）、日々の運用ログ（L3）
- 含める: 「外部プロジェクトに渡してすぐ運用が始まる」状態に必要な一式

## 警告

**L1 が空のままプロダクト価値はゼロである。** L2/L3 を肥大化させて L1 を後回しにする運用は、Product Vision に反する。L2/L3 の更新は L1 出荷可能性を高める方向にのみ正当化される。

## 現状

初期化のみ。第一弾バンドルは `web-service-team-bundle-vX` として準備中。
