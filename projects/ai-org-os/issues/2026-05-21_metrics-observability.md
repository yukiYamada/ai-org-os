# Metrics（状態監視）定義 Issue（2026-05-21）

## 背景
AI主導運用を自立的に改善するには、状態を定量的に観測できることが必要。
現状はルール・役割・提案フローはあるが、改善効果を測る指標が未定義。

## ゴール
最小5指標で運用状態を可視化し、週次で改善判断できる状態にする。

## 非ゴール
- 自動ダッシュボード実装
- 外部監視基盤導入
- リアルタイム集計

## 受け入れ条件
- [ ] 指標5つ（自立実行率/手戻り率/承認待ち時間/提案採択率/学習反映率）が定義されている
- [ ] 各指標の算出式と計測頻度（週次）が定義されている
- [ ] 初期しきい値（目標値）が定義されている
- [ ] 週次レビュー手順（記録→判定→次アクション）が定義されている

## タスク分解（30〜90分）
- [ ] M1: 指標定義と算出式を文書化
- [ ] M2: 週次記録テンプレートを `learnings.md` に追加
- [ ] M3: ワークフローに「Metrics Review」ステップを追加
- [ ] M4: backlog に定期タスクを追加

## 提案（初期しきい値）
- 自立実行率: 60%以上
- 手戻り率: 20%以下
- 承認待ち時間: 前週比 -20%
- 提案採択率: 30%以上
- 学習反映率: 50%以上
