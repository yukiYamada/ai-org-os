# 組織チャーター

> 想定読者: ai-org-os の運営構造に参加・参照する人間および AI エージェント。

## 組織名
ai-org-os Development Organization

## 存在目的
**ai-org-os というツール本体**——メンテナンス性の高い自社開発 AI チームを構築・共有可能にするツール——を開発・維持・配布する。
本組織は特定の業務ドメインに閉じない。Build / Maintain / Share の 3 軸でツール価値を提供することが存在意義である。

## スコープ
- ツールの構築（Build）: 他者が自社 AI 開発チームを立ち上げられる仕組みの整備
- ツールの維持（Maintain）: コア規約・学習ループの継続運用と自己改善
- ツールの配布（Share）: 移植可能な L1 バンドルとして成果物を出力できる状態の維持
- Git を基盤とした履歴管理・説明可能性の担保
- チーム構造・ルール・知識蓄積の維持

### L1 バンドル第一弾のリファレンス実装
`web-service` ドメイン向けチーム（`teams/web-service-team` 等）は、**L1 バンドル第一弾のリファレンス実装**として位置づける。本組織のスコープを web サービス領域に限定するものではない。

## 非スコープ（初期段階）
- 実行ランタイム（オーケストレーター・課金・UI）
- 特定 LLM ベンダーへの依存実装
- 大規模監視・自動化基盤
- マーケットプレイス機能

## 人間の責務
- 最終承認
- 説明責任
- 実行環境・権限・制約の提供
